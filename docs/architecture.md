# UX Notes

## Page Model

The prototype direction is built around a simple restaurant decision path:

1. Atmosphere: does this place feel right?
2. Menu: does the food/service match the intent?
3. Proof: do other guests trust it?
4. Booking: how quickly can I act?

## Interface Priorities

| Priority | Reason |
|---|---|
| Strong hero signal | Recruiters can see visual judgment immediately |
| Menu hierarchy | Restaurant visitors scan before they commit |
| Reservation flow | The site needs conversion intent, not just decoration |
| Responsive polish | Hospitality traffic is heavily mobile |
| Public-safe framing | The repo must not imply official ownership |

## Execution Signal

The work shows frontend taste, content structure, hospitality awareness, and the ability to prepare a public case study without exposing private working material.
